Это плагин Confluence, предназначенный для автоматического создания страниц на основе данных из .xlsx файла.
После установки плагина в интерфейсе Confluence появляется дополнительный пункт верхнего меню "Create new pages", который запускает пользовательский интерфейс плагина.
В открывшейся форме следует выбрать пространство, в котором будут созданы страницы и .xlsx файл, содержащий информацию о страницах, затем нажать кнопку "Загрузить".
Формат файла:
На первом листе два столбца с наименованиями Name и label for page.
Пример файла прилагается - example.xlsx

Поскольку у меня не было ранее опыта разработки под Confluence, я выбрал способ реализации с наиболее широкими и документированными, как мне показалось, возможностями, 
чтобы в процессе разработки не столкнуться с какими-либо ограничениями.

Преимущества реализации в виде плагина:
- не требует установки/запуска стороннего ПО ни на клиент, ни на сервер
- не требуется передача учётных данных Confluence в стороннее ПО (и вообще указывать их в коде)
- платформо-независим
- не надо распространять среди пользователей одной организации, установил один раз - пользуются все.
- есть возможность распространять через atlassian marketplace
- легко встраивается в интерфейс Confluence
- просто устанавливается
- можно расширить функционал в любом объеме
- используются рекомендуемые и поддерживаемые разработчиком функции JAVA API
- функционал JAVA API мне показался шире чем у REST API
- возможно, работает быстрее чем через REST API

Недостатки реализации:
- относительно сложная разработка и большой объем кода
